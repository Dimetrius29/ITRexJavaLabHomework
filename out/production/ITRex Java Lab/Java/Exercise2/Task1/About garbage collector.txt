Сборщик мусора (GC) позволяет автоматически находить и удалять неиспользуемые объекты, чтобы освободить память. Реализация проходит в 3 этапа: идентификация живых объектов (обход графа достижимых объектов), удаление мертвых объектов и уплотнение оставшихся живых объектов. Объекты удаляются из молодого поколения в процессе малого события сборки мусора, объекты отправляются в мусор из старшего поколения - основное событие сборки мусора. Существует 8 типов GC, их количество постепенно увеличивается с выходом новых версий jdk. Пользователь может использовать настройками по умолчанию либо изменить GC в соответствии с требованиями приложения.
